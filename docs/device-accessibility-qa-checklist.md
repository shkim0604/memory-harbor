# 실제 디바이스 접근성(UI) QA 체크리스트

Last updated (ET): 2026-03-02

## 목적
- 50대 이상 사용자 기준으로 큰 글꼴 환경에서도 핵심 UX(통화, 리뷰, 내역, 설정)가 깨지지 않는지 검증한다.
- 이번 기준은 앱 최소 텍스트 스케일 `1.2` 보장(`lib/main.dart`)을 포함한다.

## 테스트 환경
- iOS: iPhone 13/14/15 (iOS 17+)
- Android: Galaxy/AOSP 1종 이상 (Android 13+)
- 네트워크: Wi-Fi 안정망 + 4G/5G 1회씩
- 계정: caregiver 1, receiver 1

## 사전 설정
1. 앱 완전 종료 후 재실행
2. OS 글꼴 크기 단계별 테스트
3. OS 디스플레이 크기(안드로이드)도 별도 단계로 테스트

## 텍스트/디스플레이 설정 단계
1. 기본값
2. 중간 확대
3. 크게(권장 점검 기준)
4. 최대치(스트레스 테스트)

## 공통 합격 기준
1. 주요 버튼 텍스트가 잘리지 않는다.
2. 텍스트와 아이콘이 겹치지 않는다.
3. 터치 타깃이 너무 작아지지 않는다.
4. 스크롤이 필요한 영역은 정상 스크롤된다.
5. 핵심 플로우에서 오동작/크래시가 없다.

## 화면별 체크리스트

### 1) 인증/온보딩
1. 로그인 버튼 라벨이 한 줄 또는 의도된 줄바꿈으로 보인다.
2. 에러 스낵바/안내 문구가 화면 밖으로 잘리지 않는다.

### 2) 홈(케어기버)
1. 상단 공동체 이름/요약 카드의 숫자 텍스트가 겹치지 않는다.
2. 통화 기록 카드의 이름/시간/통화길이 텍스트가 서로 침범하지 않는다.
3. `전체보기` 버튼 높이와 라벨 가독성이 유지된다.

### 3) 리시버 홈/통화기록
1. `최근 통화 내역` 리스트 아이템에서 이름/일시/통화시간이 모두 표시된다.
2. 긴 이름에서도 레이아웃이 깨지지 않는다(말줄임 허용).

### 4) 통화 화면(최우선)
1. 상태 텍스트(연결중/통화중/종료)가 명확히 보인다.
2. `음소거/스피커/종료` 버튼이 한 화면에서 안정적으로 터치된다.
3. 긴 주제 문구에서도 카드 높이/버튼 영역이 무너지지 않는다.
4. 통화 연결/종료 이벤트 시 UI 갱신이 지연 없이 반영된다.

### 5) 리뷰 작성
1. 필수 항목 레이블과 입력칸이 겹치지 않는다.
2. 키보드 표시 시 입력칸 가림이 없어야 한다(스크롤 가능).
3. 제출 버튼이 항상 접근 가능한 위치에 있다.

### 6) 설정
1. 로그아웃/계정 관련 항목 라벨이 잘리지 않는다.
2. 다이얼로그 버튼 텍스트가 모두 보인다.

## 통화 시나리오 회귀 테스트
1. caregiver 발신 -> receiver 수락 -> 통화 연결 -> 종료
2. caregiver 발신 -> receiver 거절
3. caregiver 발신 -> receiver 무응답(미스드콜)
4. receiver 잠금화면 수락(가능 디바이스에서)
5. 네트워크 일시 불안정 후 복귀

검증 항목:
1. 콜 상태 전이가 기존과 동일하다.
2. 콜 종료 후 리뷰 이동/복귀 흐름이 정상이다.
3. 메모 임시저장/종료시 저장이 정상이다.
